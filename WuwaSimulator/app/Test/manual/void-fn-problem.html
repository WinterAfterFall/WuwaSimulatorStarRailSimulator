unit.rotations.set("Burst", (timeline) => {
    let t = timeline.currentFrame; // อ่าน frame ครั้งเดียว

    // schedule Ult + จบ Ult ทันที
    timeline.schedule(new AttackActionEvent("Ult", t, 90, ...));
    timeline.schedule(new NotificationEvent("Ult-end", t+90, EndAction));
    t += 90;

    // schedule BA1 + จบ BA1 ทันที
    timeline.schedule(new AttackActionEvent("BA1", t, 30, ...));
    timeline.schedule(new NotificationEvent("BA1-end", t+30, EndAction));
}); // ← push 4 events เข้า IPQ พร้อมกันทีเดียว

สรุปปัญหาทั้งหมด
หัวข้อแบบเสนอ (void fn)แบบปัจจุบัน (Queue)
ขนาด setupQueue 2 actions (Burst ของ T1, Burst ของ T2) 4 actions (Ult-T1, BA1-T1, Ult-T2, BA1-T2)
events ที่ push ต่อครั้ง 4 events ทีเดียว (ทั้ง rotation) 2 events ต่อ action (attack + end)
IPQ หลัง do-while ยังมี event ค้าง (BA1 ที่ยังไม่รัน) ว่างสนิท พร้อม action ถัดไป
Event collision T1-BA1 และ T2-Ult อยู่ที่ t=90 พร้อมกัน ไม่มี — แต่ละ action จัดการ IPQ ตัวเอง
ลำดับ action ขึ้นกับ heap internals — ไม่แน่นอน ควบคุมได้จาก Queue order
Loop rotation ต้องคิด mechanism ใหม่ทั้งหมด rotate() ใน Queue จัดการได้ทันที
🔑 ต้นตอปัญหาหลัก
void fn schedule ทั้ง rotation พร้อมกัน ก่อนที่ action แรกจะจบ
ระบบ lock ของ RotationDirector ออกแบบมาให้ทำงาน ทีละ action
สองสิ่งนี้ขัดกันโดยพื้นฐาน — แก้ไม่ได้โดยไม่เปลี่ยน architecture
